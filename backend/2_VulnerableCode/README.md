# VulnerableCode

이 폴더는 수업에서 `backend/routes`의 보안 코딩 버전과 비교하기 위한 취약 코드 예시입니다.
실제 서버에서 `require`하거나 라우팅에 연결하지 마세요.
주석에 적힌 공격 방식은 반드시 로컬 수업 환경에서만 설명하고 재현하세요.

## 비교 포인트

- SQL 파라미터 바인딩을 쓰지 않고 문자열 보간으로 쿼리를 조립합니다.
- 숫자, 페이지 크기, 검색어, 파일명 같은 입력값을 검증하지 않습니다.
- 클라이언트가 보낸 권한값이나 작성자명을 신뢰합니다.
- 업로드 파일 개수, 크기, 확장자, 저장 파일명을 제한하지 않습니다.
- 세션 종료, 인증 확인, 에러 처리 같은 흐름이 불완전합니다.

## 파일 구성

- `routes/auth.js`: 로그인, 회원가입, 내 정보, 로그아웃의 취약 구현
- `routes/login-check.js`: 권한 검사를 클라이언트 입력에 의존하는 취약 미들웨어
- `routes/notice.js`: 공지 목록 조회의 취약 구현
- `routes/user-record.js`: 사용자 전적 조회의 취약 구현
- `routes/community.js`: 게시판 목록, 상세, 작성, 첨부 다운로드의 취약 구현
- `routes/admin/user.js`: 관리자 사용자 조회/수정의 취약 구현
- `routes/datas/upload-csv.js`: CSV 업로드의 취약 구현
